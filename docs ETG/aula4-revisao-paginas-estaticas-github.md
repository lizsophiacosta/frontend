\# Páginas Estáticas no GitHub



\## O que são páginas estáticas

Páginas estáticas são sites compostos por arquivos simples como HTML, CSS e JavaScript, sem processamento no servidor. Todo o conteúdo é fixo e exibido diretamente ao usuário.



\## GitHub Pages

O GitHub Pages é um recurso do GitHub que permite hospedar páginas estáticas diretamente de um repositório, de forma gratuita.



\## Como funciona

O GitHub utiliza os arquivos do repositório para gerar e disponibilizar um site na internet. Normalmente, o site é publicado a partir de uma branch específica (como `main`) ou de uma pasta como `/docs`.



\## Como criar uma página estática



1\. Criar um repositório no GitHub

2\. Adicionar um arquivo `index.html`

3\. Inserir o conteúdo da página usando HTML, CSS e JavaScript

4\. Fazer commit e push das alterações

5\. Acessar as configurações do repositório (Settings)

6\. Ativar o GitHub Pages na seção "Pages"

7\. Escolher a branch e pasta (root ou /docs)

8\. Salvar e aguardar a publicação



\## Estrutura básica



Exemplo de um arquivo `index.html`:



<!DOCTYPE html>

<html>

<head>

&#x20;   <title>Meu Site</title>

</head>

<body>

&#x20;   <h1>Olá, mundo!</h1>

</body>

</html>



\## Vantagens



\- Hospedagem gratuita

\- Fácil de configurar

\- Ideal para portfólios e projetos simples



\## Limitações



\- Não suporta backend

\- Conteúdo não é dinâmico



\## Resumo

O GitHub Pages permite criar e publicar páginas estáticas de forma simples, usando apenas arquivos básicos da web.
